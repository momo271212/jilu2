## 🔧 解决方案

### 方案 1：使用个人访问令牌（PAT）⭐ 推荐

#### 步骤 1：创建 GitHub Token
1. 访问 https://github.com/settings/tokens
2. 点击 **"Generate new token (classic)"**
3. 设置：
   - **Note**: `Push to jilu2`
   - **Expiration**: 7 days (或自定义)
   - **Scopes**: 勾选 `repo` (完整仓库访问)
4. 点击 **"Generate token"**
5. **复制生成的令牌**（只显示一次！）

#### 步骤 2：配置 Git 使用 Token
```bash
cd "C:\Users\lenovo\Desktop\记录软件"
git remote set-url origin https://YOUR_TOKEN@github.com/momo271212/jilu2.git
```

替换 `YOUR_TOKEN` 为你刚生成的令牌。

#### 步骤 3：推送
```bash
git push origin main
```

### 方案 2：配置 SSH 密钥

#### 步骤 1：生成 SSH 密钥
```bash
ssh-keygen -t ed25519 -C "a13256836835@qq.com"
# 按 Enter 使用默认路径
# 输入密码（可选）
```

#### 步骤 2：添加公钥到 GitHub
```bash
# 复制公钥
cat ~/.ssh/id_ed25519.pub
# 或 Windows:
type %USERPROFILE%\.ssh\id_ed25519.pub
```

1. 访问 https://github.com/settings/keys
2. 点击 **"New SSH key"**
3. 粘贴公钥内容
4. 保存

#### 步骤 3：更改远程地址为 SSH
```bash

```

#### 步骤 4：推送
```bash

```

## ⚠️ 常见问题

### Q1: Permission denied (publickey)
**原因**: SSH 密钥未配置或未添加到 GitHub
**解决**: 使用方案 1 (PAT) 或重新配置 SSH

### Q2: remote: Repository not found
**原因**: 仓库地址错误或你没有权限
**解决**: 确认仓库地址正确，或请求仓库所有者添加你为 collaborator

### Q3: Everything up-to-date
**原因**: 本地已经和远程同步
**解决**: 检查 `git status`，可能需要 `git pull` 先

### Q4: 连接超时
**原因**: 网络问题，无法访问 GitHub
**解决**:
- 检查网络连接
- 配置代理（如需要）
- 使用 GitHub 镜像
